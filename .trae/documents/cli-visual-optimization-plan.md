# CLI 输出视觉优化计划

## 摘要

对 ETF Analyzer 命令行输出的颜色和排版进行优化，使"涨跌幅"、"收益率"等关键指标按中国金融行业习惯（正值红色、负值绿色）着色，并使用横线/竖线统一排版对齐。

## 当前状态分析

* `main.py` 中 `ETFCLI` 类负责所有命令行交互输出

* 当前所有 `print()` 输出均为终端默认白色，无颜色区分

* 排版使用 `_print_separator(char="=", length=40)` 为唯一分隔方式，无结构化排版设计

* 各数据项以单行 `print(f"  {key}: {value}")` 形式输出，数字、文字混排，对齐依赖缩进

## 修改方案

### 修改 1：新增颜色工具函数 —— `main.py`

在 `ETFCLI` 类中添加静态方法：

```python
# ANSI 颜色码
_COLOR_RED = "\033[91m"    # 红色 - 正值（中国金融惯例：涨=红）
_COLOR_GREEN = "\033[92m"  # 绿色 - 负值（中国金融惯例：跌=绿）
_COLOR_RESET = "\033[0m"   # 重置颜色
_COLOR_CYAN = "\033[96m"   # 青色 - 标题/标签
_COLOR_YELLOW = "\033[93m" # 黄色 - 警告/中性值

@staticmethod
def _color_value(value, is_percentage=True):
    """根据数值正负返回带颜色的字符串。

    正值显示红色，负值显示绿色，零值显示默认色。
    若 value 非数值或无法解析，返回原字符串。
    """
```

### 修改 2：科学计数法数值格式化 —— `main.py`

当前部分指标（如 `daily_volatility` 日波动率、`tracking_error` 跟踪误差等）为极小数值，输出时会出现 `0.000123` 甚至科学计数法 `1.23e-04`，可读性差。

新增静态方法：

```python
@staticmethod
def _format_number(value, decimals=4, as_percentage=False):
    """格式化数值，消除科学计数法，按精度显示。

    将科学计数法（如 1.23e-04）转为常规小数（0.0001），
    对百分比类数值保留 2 位小数，对波动率/误差类保留 4-6 位。

    Args:
        value: 数值或字符串
        decimals: 小数位数
        as_percentage: 是否以百分比形式显示

    Returns:
        str: 格式化后的数字字符串
    """
```

受影响的输出点：

* `daily_volatility`（日波动率）→ 格式化到 6 位小数

* `annualized_volatility`（年化波动率）→ 格式化到 4 位小数

* `max_drawdown`（最大回撤）→ 百分比格式 2 位小数

* `tracking_error`（跟踪误差）→ 格式化到 4 位小数

* `sharpe_ratio`（夏普比率）→ 格式化到 4 位小数

* `information_ratio`（信息比率）→ 格式化到 4 位小数

* `annualized_return`（年化收益率）→ 百分比格式 2 位小数

* `excess_return`（超额收益）→ 百分比格式 2 位小数

* `win_rate`（胜率）→ 百分比格式 2 位小数

### 修改 4：布局统一优化 —— 整个 `main.py`

* **水平分隔线**：区分不同数据区块时用 `=` 分隔线（区块间），区块内次级分隔用 `-`

* **竖线对齐**：每一区块内的 key-value 输出用 `|` 对齐排列，统一格式为   `  {key} | {value}`

* 所有日期/ETF代码等标签使用 `_COLOR_CYAN` 高亮

* 错误/警告信息使用 `_COLOR_YELLOW`

### 修改 5：ETF 列表输出表格化 —— `_fetch_etf_list`

当前使用 `df.to_string(index=False)` 输出，保持该方式但在表头添加列对齐标识。

2. 测试数据获取各子菜单颜色是否按正负显示
3. 测试数据分析各子菜单颜色是否正确

5. 运行 `pytest tests/ -v` 确保无回归

## 假设与决策

* 颜色实现使用 ANSI 转义码，无需额外依赖

* Windows 终端默认支持 ANSI 颜色（PowerShell 5+ / Windows Terminal）

* 中国金融行业色彩惯例：涨/正值=红色，跌/负值=绿色

* 中文字体在终端中等宽显示，竖线对齐使用空格填充保证视觉效果
